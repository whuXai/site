A4-Accessibility无障碍设计

发表于 2019-08-24 | 更新于 2019-09-04 | 分类于 UX 学习课程

好的无障碍设计是网站或应用程序成功的一个关键，不仅因为无障碍设计在许多国家是法律明文规定的，还要考虑到上百万、千万的各类不便用户，这个数字随着老龄化还在增加。不管出于道德和法律上的原因，还是为了让你的产品能够充分发挥潜能，都应该做好无障碍设计。
本章节将从各个角度了解无障碍设计，知道设计决策如何影响无障碍和可用性；应该遵循的无障碍的标准；如何将无障碍的纳入设计和开发过程，以及如何评估和验证。

有关Accessibility，有几个不错的系列：
首先是W3C的WAI及其解释说明的部分；
然后是WebAIM，斯坦福的SOAP，伯克利的总结；
或者看Google的开发者文档，IBM的规范，Material的指引和游戏方面的Guidelines。

IDF课程本质上也是抽取汇总了各个系列的部分内容。

一， 基础背景

IDF上这课的导师是Frank Spillers，应该是他的培训课程

1，Spillers解释过why it matters，从Legal、Moral和Business的角度；嫌视频太短的也可以看一篇文章。
2，不同Disability类型对应不同的无障碍设计考量，对应文章，延伸阅读，几张海报；
3，无障碍设计对于可用性和Social Needs的影响，还有早期的研究论文。
4，无障碍设计对于SEO的影响，对应文章；
5，无障碍设计的ROI，参考文章。

二，合规知识

1，ADA（美国残疾人法案）的知识点可以搜索了解下， 官网很多页面打不开。
2，Section 508的相关内容也可以看看，还有美国政府网站的指导文件，以及如何应用到网站设计。
3，建议参考的WCAG规范如何理解规范，以及相关的一个Q&A。
4，注意WCAG规范中的合规要求：A，AA，AAA，以及如何理解。

三，设计原则

第一部分：
1，可以借鉴通用设计的原则，对应文章；
2，也可以参考Design for ALL，或者Usability for ALL；
3，参考WAI的设计方法，对应文章，或者看这篇文章。
4，Web和移动端无障碍设计的10个原则，对应文章；

第二部分：
5，地图类应用的考量点，我没有找到资料，可以看看这个视频；
6，社交媒体的无障碍设计考量，参考文章，或者这篇，或者这篇，还有这个report；
7，移动端无障碍设计，还可以参考WAI的规范，iOS的建议和Android指引；
8，VR/AR的无障碍设计考量，可以参考IBM的观点，游戏设计师的建议和墨尔本大学的分析。

四，优化代码

1，布局、导航、图片、表单、图表、搜索和焦点控制方面，可以看WAI的部分内容；
2，键盘输入、弹窗和框架，动态内容如信息图、动画、音频、视频、动态事件，第三方内容方面，可以看WebAIM的内容；
3，HTML5、CSS、ARIA Markup、Javascript方面，可以看AIRA的部分，或者华盛顿大学的一个指引。

也许看Google的开发者文档就够了。

五，测试评估

最后一节课，是关于评估和测试，W3C的WAI部分基本满足需求，测试什么、测试工具，如何测试等等。
